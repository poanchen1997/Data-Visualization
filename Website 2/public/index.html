Effectiveness of Tree-Maps as Tree Visualization Techniques

Team Members: Po An Chen, Yiling Chen, Coco Luo

Motivation

Tree Structure visualization techniques are one of the important techniques to support the analysis of large hierarchical structure data. We usually use them as tree structure node-link diagram. However, it is difficult to visualize the content of directories when there are too many child nodes. In this project, we studied the effectiveness of the treemaps as tree structure visualization. Throughout the evolution of treemaps visualization techniques, different people developed different kinds of treemap sysmtems. Only four treemaps will be discussed here which are: standard treemaps, circular treemaps, Voronoi Treemaps, and Geo-Voronoi Treemaps.

Treemap



Credit: Treemap, CSV by Mike Bostock
Introduced by Ben Shneiderman, a treemap is an ideal tool for displaying large amounts of hierarchically structured data. The space in the visualization is split up into rectangles that are sized and ordered by a quantitative variable. This treemap is built to describe the class hierarchy for the Flare ActionScript visualization library. The dataset contains both the class hierarchy as well as the import connections between classes. Various algorithms can be used to determine how the rectangles in this treemap to be sized and ordered. From the treemap contributed by Bostock, we want to show that the viewers can select any shape of the tile they prefer to fit their data: binary, square, dice, etc.
This treemap was created by Plotly using the COVID-19 dataset. It presents the total number of COVID cases in different countries from 2021 and 2022. The levels in the hierarchy of the treemap are visualized as rectangles containing other rectangles. Continents are displayed as large rectangles, where countries are small rectangles nested inside those large ones. Each set of rectangles on the same level in the hierarchy represents a column in a data table. Each rectangle on a level in the hierarchy represents a category in a column. Colors and sizes are the most important encodings as they can become the factor for fast and accurate decision-making. Here, different colors are used to represent different continents, readers can see the proportions of COVID cases compared across years and among countries through the sizes of the rectangles. When the readers move their mouse over to one rectangle, a text box would show up and provide information about the parent node"continent", the child node"country", and the "total cases" in the selected country. By clicking their mouse, the readers can also zoom in and navigate to the selected region.

Voronoi Treemap




Voronoi treemap as an alternative to traditional rectangular treemaps for visualizing hierarchical data. Like rectangular treemaps, Voronoi treemaps represent hierarchical data by dividing the canvas region into cells for each node at the top of the hierarchy, and then further dividing each of these cells for the children of those nodes. The organic shapes created by the Voronoi treemap can be easier to distinguish sibling nodes from nodes in other branches of the hierarchy. Voronoi treemaps can also be fit to non-rectangular canvases, and are often more aesthetically pleasing.

Zoomable Circle Packing



Credit: Zoomable Circle Packing by Mike Bostock



In circular treemaps, nested circles will be used to show the tree structure. Instead of using rectangular areas to represent a directory hierarchical data, here we used a circular treemap to describe the class hierarchy for the Flare ActionScript visualization library. The dataset we used contains both the class hierarchy as well as the import connections between classes. From the visualization, we can see that the size of each child is proportional to the number of its content. The viewers can zoom in and out by clicking on a circle. A circular treemap looks more pleasing than a standard treemap and it shows the hierarchies clearer. However, this type of tree structure techniques are not acceptable to some users due to the many circles of the nodes.

Concusion and Evaluation

This table summarizes the advantages and disadvantages of the four different types of tree-maps. These results were initially gathered during literature and were then supported by us fitting those plots on various data. Treemaps are highly efficient space since it uses space-filling techniques to represent data in a rectangular area. This makes treemaps suitable for large hierarchical structures and those trees with high aspect ratios on branches. Besides that, the space efficiency of treemaps allows additional graphical data to be combined into whole representation of a tree structure. Scalability is an exponential growth of treemaps. It shows how easy it is to expand the treemaps to become more effective tree visualization techniques. One main disadvantages of treemaps are low visibility. Users have difficulty seeing the whole hierarchical structure. It is more difficult to perceive the different levels of the tree. In can be frustrated to some users due to its difficulty to view the overall structure.
Tree-maps provide users with an overview of a large and complex tree structure with a quicker processing time than the traditional node-link diagram. The four different types of tree-maps will give users a choice to choose the best representation that suits their application. All type of treemaps has their own advantages and also disadvantages. The main important elements of tree-maps are efficiency space and scalability. Almost all types of treemaps make use of its space efficiently by not leaving any free space when visualizing a tree structure. These efficient uses of spaces lessen the need for panning and zooming. The scalability of various tree-maps in high when representing large hierarchical structure data sets. While for a small data sets, it might seem to be not to important to use treemaps. Tree representation are common structure of data representation, we believe that tree-maps will continue to evolve and could one day become one of the main method in data visualization.
